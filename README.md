# Development Documentation

## Development Progress

The development of the Student Admission Application System began by setting up the project using Flutter and connecting it to Firebase. This created the foundation needed to build the application's features and manage user information securely.

The first stage of development focused on designing the screens that users see when they launch the application. A splash screen was created to display the application's branding while the system loads. This was followed by a welcome screen that introduces users to the application before they proceed.

The next step was implementing the user authentication system. A sign-in screen was developed to allow existing users to access their accounts securely. In addition, a "Forgot Password" screen was created to help users reset their passwords whenever they are unable to log in.

Once the authentication process was completed, development moved to the main part of the application. A bottom navigation bar was implemented to allow users to move easily between the four main sections of the application:

- Home
- Schools
- Applications
- Profile

The Home screen was then developed to serve as the user's dashboard. It provides a simple overview of important information and quick access to frequently used features.

A **Quick Actions** section was added to the Home screen to make important tasks easier to access. At this stage, users can quickly navigate to:

- Find Schools
- My Documents

To help users monitor their admission activities, a **My Applications** summary section was also developed. This section displays the current status of all applications by showing:

- Total Applied
- Under Review
- Accepted
- Rejected

This allows users to view the progress of their applications at a glance without opening each application individually.

To improve the appearance of the dashboard and make it more engaging, a scrolling **Featured Schools** banner was added. This section highlights selected schools and introduces users to institutions they may be interested in exploring.

Throughout the development process, Flutter and Dart have been used to build the application's interface and functionality, while Firebase has been used to support user authentication and manage application data.

At the current stage of development, the foundational structure of the application has been completed successfully. Users can securely access the application, navigate between its main sections, and view an organized dashboard containing important admission-related information.

The next phase of development will focus on implementing additional features, including:

- School listing and search
- School details
- Admission application forms
- Document upload
- Application tracking
- Payment integration
- Profile management
- Additional system improvements
